RealSense + Isaac ROS + NITROS Setup
1. Install system dependencies on the host
Update apt and install the packages needed to build librealsense from source:

sudo apt update
sudo apt install -y \
  git cmake build-essential pkg-config \
  libssl-dev libusb-1.0-0-dev libgtk-3-dev \
  libglfw3-dev libgl1-mesa-dev libglu1-mesa-dev \
  at libudev-dev python3-dev python3-pip


2. Install the RealSense udev rule on the host
Download the RealSense USB rule and place it in the udev rules directory so the cameras can be accessed correctly:

cd ~
wget https://raw.githubusercontent.com/IntelRealSense/librealsense/v2.55.1/config/99-realsense-libusb.rules
sudo mv 99-realsense-libusb.rules /etc/udev/rules.d/
sudo udevadm control --reload-rules
sudo udevadm trigger


3. Create the host ROS 2 workspace
We will use a host workspace for librealsense and realsense-ros:

mkdir -p ~/ros2_ws/src
cd ~/ros2_ws


4. Clone and build librealsense on the host
Clone the official Intel RealSense SDK source and build version v2.55.1:

cd ~/ros2_ws
git clone https://github.com/IntelRealSense/librealsense.git
cd librealsense
git checkout v2.55.1
mkdir -p build
cd build
cmake .. \
  -DCMAKE_BUILD_TYPE=Release \
  -DFORCE_RSUSB_BACKEND=ON \
  -DBUILD_EXAMPLES=true \
  -DBUILD_GRAPHICAL_EXAMPLES=true \
  -DBUILD_UNIT_TESTS=false \
  -DBUILD_PYTHON_BINDINGS=false
make -j"$(nproc)"
sudo make install
sudo ldconfig

Important points:
* FORCE_RSUSB_BACKEND=ON is important
* BUILD_UNIT_TESTS=false avoids test-related issues
* the version used here is v2.55.1

5. Clone and build realsense-ros on the host
realsense-ros must be built on the host inside ~/ros2_ws, not inside the Isaac ROS Docker workspace.

cd ~/ros2_ws/src
git clone -b ros2-master https://github.com/IntelRealSense/realsense-ros.git

Build it on the host:

cd ~/ros2_ws
source /opt/ros/humble/setup.bash
rosdep update
rosdep install --from-paths src --ignore-src -r -y
colcon build --symlink-install --packages-select realsense2_camera_msgs
source ~/ros2_ws/install/setup.bash
colcon build --symlink-install --packages-select realsense2_camera
source ~/ros2_ws/install/setup.bash


6. Test RealSense on the host
First test the SDK:

realsense-viewer

Then confirm the ROS 2 driver is available:

source /opt/ros/humble/setup.bash
source ~/ros2_ws/install/setup.bash
ros2 pkg list | grep realsense2_camera


7. Prepare the Isaac ROS workspace
Now create the Isaac ROS workspace for the NVIDIA packages and your own packages:

sudo systemctl daemon-reload && sudo systemctl restart docker
sudo apt-get install -y git-lfs
git lfs install --skip-repo

mkdir -p ~/workspaces/isaac_ros-dev/src
echo "export ISAAC_ROS_WS=${HOME}/workspaces/isaac_ros-dev" >> ~/.bashrc
source ~/.bashrc

Add your user to the docker group if needed:

sudo usermod -aG docker $USER
newgrp docker
groups


8. Clone the Isaac ROS and project repositories
Clone the needed repositories into ${ISAAC_ROS_WS}/src.
Do not clone realsense-ros here again for building. It belongs to ~/ros2_ws on the host.

cd ${ISAAC_ROS_WS}/src

git clone https://github.com/DhiaNeifar/Realsense-IsaacROS.git


9. Configure and enter the Isaac ROS dev container
Create the Isaac ROS container config:

cd ${ISAAC_ROS_WS}/src/isaac_ros_common/scripts
touch .isaac_ros_common-config
echo CONFIG_IMAGE_KEY=ros2_humble.realsense > .isaac_ros_common-config

Launch the Isaac ROS dev environment:

cd ${ISAAC_ROS_WS}/src/isaac_ros_common
./scripts/run_dev.sh -d ${ISAAC_ROS_WS}

Once inside the container, the workspace path will be:

/workspaces/isaac_ros-dev


10. Install dependencies inside the Isaac ROS container
Inside the Isaac ROS Docker container:

cd /workspaces/isaac_ros-dev
source /opt/ros/humble/setup.bash

sudo apt-get update
rosdep update
rosdep install --from-paths src --ignore-src -r -y

If apt metadata is stale and dependency installation fails, clean and refresh apt first:

sudo apt-get clean
sudo rm -rf /var/lib/apt/lists/*
sudo apt-get update
rosdep update
rosdep install --from-paths src --ignore-src -r -y


11. Build Isaac ROS packages, including isaac_ros_image_proc
When building in Isaac ROS, do not stop at only isaac_ros_nitros. You must also build up to isaac_ros_image_proc, since the RectifyNode comes from that package.

cd /workspaces/isaac_ros-dev
source /opt/ros/humble/setup.bash
colcon build --packages-up-to isaac_ros_nitros --symlink-install --cmake-args -DBUILD_TESTING=OFF
source /workspaces/isaac_ros-dev/install/setup.bash

colcon build --packages-up-to isaac_ros_image_proc --symlink-install --cmake-args -DBUILD_TESTING=OFF
source /workspaces/isaac_ros-dev/install/setup.bash

You can verify the package and component are visible:

ros2 pkg list | grep isaac_ros_image_proc
ros2 component types | grep -i Rectify


12. Fix NITROS temporary graph directory permissions
If NITROS fails with an error like:

filesystem error: cannot create directories: Permission denied [/tmp/isaac_ros_nitros/...]

then fix the temporary directory permissions inside the container:

sudo rm -rf /tmp/isaac_ros_nitros
sudo mkdir -p /tmp/isaac_ros_nitros
sudo chmod -R 777 /tmp/isaac_ros_nitros

You can also inspect the permissions with:

ls -ld /tmp /tmp/isaac_ros_nitros


13. Start the NITROS component container
In a shell inside the Isaac ROS container:

cd /workspaces/isaac_ros-dev
source /opt/ros/humble/setup.bash
source /workspaces/isaac_ros-dev/install/setup.bash
ros2 run rclcpp_components component_container_mt --ros-args -r __node:=nitros_rectify_container


14. Load the RectifyNode component
In another shell inside the same Isaac ROS container:

cd /workspaces/isaac_ros-dev
source /opt/ros/humble/setup.bash
source /workspaces/isaac_ros-dev/install/setup.bash

Example for the D405 color stream:

ros2 component load /nitros_rectify_container isaac_ros_image_proc nvidia::isaac_ros::image_proc::RectifyNode \
  --node-name d405_rectify \
  -p output_width:=640 \
  -p output_height:=480 \
  -r image_raw:=/d405/camera/color/image_raw \
  -r camera_info:=/d405/camera/color/camera_info \
  -r image_rect:=/d405/camera/color/image_rect \
  -r camera_info_rect:=/d405/camera/color/camera_info_rect
